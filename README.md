# 飞书每日销售看板 AI Agent 工作流

这是一个生产级的、基于 Python 开发的 AI Agent 自动化报表管道。该系统可自动从 **飞书多维表格 (Bitable)** 拉取原始交易记录，调用大语言模型 (LLM) 进行深度指标提炼、异常标红与趋势分析，并将格式化好的高大上交互式飞书消息卡片自动推送到 **飞书群机器人 Webhook**。

针对生产环境可能遇到的接口限流、网络偶发抖动、空数据、大模型“数值与幻觉”等稳定性痛点，本项目设计并实现了完备的 **反思自我纠错环路 (Reflection Loop)**、**网络退避重试** 与 **降级容灾兜底 (Fallback)** 机制，确保管道高可用。

## 架构拓扑与数据流

```
+------------------+
|  飞书多维表格    |
+--------+---------+
         | (拉取原始交易记录)
         v
+------------------+
|  飞书 API 客户端  | <--- [网络指数退避重试 (Tenacity): 自动重连 429/5xx 错误]
+--------+---------+
         | (格式校验 / 空值检查)
         v
+------------------+
|   智能体工作流   | <--- [若数据为空: 直接发送空白预警卡片并终止]
+--------+---------+
         |
         | (动态构建 Prompt 并注入数据上下文)
         v
+------------------+
|     LLM 调用     | <--- [生成结构化 JSON 每日报表]
+--------+---------+
         |
         v
+------------------+          是
| 数据 guardrails  +--------------------------------------------+
| 规则代数强校验   |                                            |
+--------+---------+                                            |
         |                                                      |
         | 否 (数据错乱、求和偏差、幻觉字段)                     |
         v                                                      v
+-----------------------+                               +---------------+
|     反思纠错环路      |                               | 发送飞书卡片  |
|                       |                               | (交互式卡片)  |
| - 捕获异常 Trace      |                               +---------------+
| - 重新组织 Prompt 反传|
| - 设定反思上限 (Max)  |
+--------+--------------+
         |
         | (若多次反思后依然计算错误)
         v
+-------------------------+
|   程序化降级兜底计算    | (使用 Python 代数算法进行硬计算，确保报表基础数据绝对准确)
+-------------------------+
```

## 快速开始

### 1. 安装依赖
确保已安装 Python 3.8+，然后安装核心依赖包：
```bash
pip install -r requirements.txt
```

### 2. 配置环境变量 (.env)
在项目根目录下创建一个 `.env` 文件（可参考以下模板。如留空，系统将自动进入**免密钥本地仿真模拟模式**，非常便于测试与汇报）：
```env
# 飞书 API 配置 (留空则使用飞书数据模拟器)
FEISHU_APP_ID=
FEISHU_APP_SECRET=
FEISHU_BITABLE_APP_TOKEN=
FEISHU_BITABLE_TABLE_ID=
FEISHU_WEBHOOK_URL=

# 大模型 API 配置 (留空则自动开启 LLM 仿真器模拟幻觉/自愈)
LLM_API_KEY=
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini

# 弹性调优参数
MAX_RETRIES=3
MAX_REFLECTION_ATTEMPTS=3
```

## 稳定性与边界场景测试

我们设计了涵盖 5 大生产级边界 Case 的自动化测试套件。您可以通过以下命令运行测试，查看智能体是如何自愈和兜底的：

```bash
python run_tests.py
```

### 测试场景详解：
1. **Happy Path (正常业务流程)**：测试在网络与大模型均正常时，报表拉取、生成、Pydantic 校验和格式化推送的完整闭环。
2. **Boundary Case - 空数据警告**：测试飞书多维表格无今日数据时，工作流自动拦截、免除大模型调用，并自动发布灰色无数据看板的机制。
3. **Boundary Case - 大模型幻觉自我纠错 (Self-Correction)**：模拟大模型算错总额或捏造虚假销售员的情况。校验引擎自动比对 Python 代数与大模型 JSON 指标，捕获偏差数据，组装反馈 Prompts，并验证大模型在第二轮迭代中自愈改正。
4. **Boundary Case - API 网络抖动与限流**：模拟飞书或大模型服务端返回 503 等瞬时错误。Tenacity 拦截请求，运行指数退避策略睡眠重试，并在网络恢复后成功拉取数据。
5. **LLM 连续校验失败的硬降级容灾**：模拟大模型连续 3 次反思纠错依然失败。工作流自动中断迭代，启动纯 Python 统计算法，组装出正确的结构化报表并标记“程序化容错计算”黄色警报，保证数据 100% 精确与业务不中断。
